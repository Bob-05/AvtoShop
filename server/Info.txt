5. Документация API
Базовый URL
text
http://localhost:5000/api
Эндпоинты
1. Авторизация
Метод	Эндпоинт	Заголовки	Тело запроса	Ответ
POST	/login	Content-Type: application/json	{"email": "admin@avtoshop.ru", "password": "admin123"}	{"success": true, "token": "JWT"}
Коды ответов:

200 — успешный вход

400 — отсутствует email или пароль

401 — неверные учетные данные

500 — ошибка сервера

2. Услуги (публичные)
Метод	Эндпоинт	Описание
GET	/posts	Получить список всех услуг
Пример ответа:

json
[
  {
    "id": 1,
    "name": "Замена масла",
    "description": "Полная замена моторного масла и фильтра",
    "price": "от 500 ₽",
    "icon_url": "/uploads/oil.png",
    "created_at": "2026-04-14T10:52:13.000Z",
    "updated_at": "2026-04-14T10:52:13.000Z"
  }
]
3. Услуги (защищённые, только для админа)
Требуется заголовок: Authorization: Bearer <JWT-токен>

Метод	Эндпоинт	Тело запроса	Описание
POST	/posts	{"name": "...", "description": "...", "price": "...", "icon": "..."}	Добавить услугу
PUT	/post/:id	{"name": "...", "description": "...", "price": "...", "icon": "..."}	Обновить услугу
DELETE	/post/:id	—	Удалить услугу
Коды ответов:

201 — услуга создана

200 — услуга обновлена / удалена

400 — отсутствуют обязательные поля

401 — отсутствует токен

403 — токен недействителен или недостаточно прав

404 — услуга не найдена

500 — ошибка сервера

4. Отзывы (публичные)
Метод	Эндпоинт	Тело запроса	Описание
GET	/reviews	—	Получить список всех отзывов
POST	/reviews	{"name": "...", "text": "...", "rating": 5}	Добавить отзыв
Пример ответа GET:

json
[
  {
    "id": 1,
    "name": "Алексей Петров",
    "text": "Отличный сервис!",
    "rating": 5,
    "created_at": "2026-04-14T10:52:13.000Z"
  }
]
Коды ответов:

200 — успешное получение

201 — отзыв создан

400 — отсутствуют обязательные поля

500 — ошибка сервера

6. Перенос данных из файлов в БД
Ранее данные об услугах и отзывах хранились статически в JavaScript-файлах. После реализации backend-модулей все данные перенесены в базу данных MySQL.

6.1. SQL-скрипт с начальными данными
Тестовые данные добавлены через SQL-запросы (см. Практическую работу №7):

admin: 1 запись (admin@avtoshop.ru)

services: 8 записей (услуги автосервиса)

reviews: 8 записей (отзывы клиентов)

6.2. Проверка переноса данных
После запуска сервера данные загружаются из БД через API:

bash
# Проверка услуг
curl http://localhost:5000/api/posts

# Проверка отзывов
curl http://localhost:5000/api/reviews
